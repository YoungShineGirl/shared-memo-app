# Level 1: 项目架构和最基础知识

适合对象：几乎没有前端或后端基础，只想知道这个项目是怎么组成、怎么工作的学生。

## 这个项目是什么

这是一个共享记事本网页。

用户可以在网页上：

- 新增备忘录
- 查看备忘录
- 编辑备忘录
- 删除备忘录

这个网页部署到公网后，手机、电脑、平板都可以打开同一个网址。所有人看到的是同一份备忘录数据。

公网地址示例：

```text
https://shared-memo-app.onrender.com
```

## 它为什么叫“共享”

如果只是普通网页，数据可能只保存在某个用户自己的浏览器里。

这个项目不是这样。它的数据保存在服务器上：

```text
用户 A 的手机
用户 B 的电脑
用户 C 的平板
      |
      v
同一个后端服务器
      |
      v
同一份备忘录数据
```

所以，一个人在手机上新增备忘录，另一个人在电脑上刷新页面后也能看到。

## 项目由哪些部分组成

这个项目主要有三部分：

```text
前端页面
后端服务器
数据文件
```

### 前端页面

前端就是用户在浏览器里看到和操作的部分。

本项目的前端文件在：

```text
public/index.html
public/style.css
public/app.js
```

它们分别负责：

```text
index.html  页面结构，比如标题、按钮、输入框
style.css   页面样式，比如白色背景、按钮样式、弹窗样式
app.js      页面交互，比如点击保存、点击删除、加载列表
```

### 后端服务器

后端是运行在服务器上的程序。用户看不到它，但浏览器会向它发送请求。

本项目的后端文件是：

```text
server.js
```

后端负责：

```text
接收浏览器请求
判断用户想做什么
调用数据功能
把结果返回给浏览器
```

### 数据文件

备忘录需要保存下来，否则刷新网页后就没了。

本项目使用一个 JSON 文件保存数据：

```text
data/memos.json
```

这个文件会在服务器运行时自动创建。

## 浏览器和服务器怎么沟通

浏览器通过 HTTP 请求和服务器沟通。

可以先把它理解成这样：

```text
浏览器：我要所有备忘录
服务器：好的，这是所有备忘录

浏览器：我要新增一条备忘录
服务器：好的，已经保存

浏览器：我要删除第 3 条备忘录
服务器：好的，已经删除
```

这些“请求地址”叫 API。

本项目有这些 API：

```text
GET    /api/memos       获取所有备忘录
GET    /api/memos/:id   获取一条备忘录
POST   /api/memos       新增备忘录
PUT    /api/memos/:id   修改备忘录
DELETE /api/memos/:id   删除备忘录
```

## 一个新增备忘录的过程

用户点击保存时，发生了这些事情：

```text
1. 用户在浏览器输入备忘录内容
2. 用户点击“保存”
3. public/app.js 发送请求给后端
4. server.js 收到请求
5. server.js 调用 database.js 保存数据
6. 数据写入 data/memos.json
7. 后端告诉前端保存成功
8. 前端重新加载备忘录列表
```

## 为什么需要部署

本地运行时，网址通常是：

```text
http://localhost:3000
```

这个地址主要给自己的电脑使用。

如果想让其他人从广域网访问，就要部署到云平台，比如 Render。

部署后会得到公网网址：

```text
https://shared-memo-app.onrender.com
```

这样任何人只要有网络和浏览器，就可以打开这个网页。

## 这个项目能学到什么

通过这个项目，可以理解网页工程最基础的几个概念：

```text
HTML 负责页面结构
CSS 负责页面样式
JavaScript 负责页面交互
Node.js 运行后端程序
Express 帮我们创建服务器和 API
JSON 文件保存数据
Render 把项目部署到公网
GitHub 保存代码
```

## 最重要的一句话

这个项目的核心架构是：

```text
浏览器里的网页
通过 API 请求
访问服务器
服务器读写数据
再把结果返回给网页
```

这就是很多网页应用最基础的工作方式。
